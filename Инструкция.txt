Задание:

Вы получили макет экрана от дизайнера. Ваша задача — сверстать экран так, чтобы он был:
1. адаптивным — чтобы один yaml файл подходил под разные разрешения экранов,
2. гибким и переиспользуемым — чтобы компоненты интерфейса, шрифты и цвета не дублировались и их можно было использовать где-нибудь еще,
3. связанным с игровой логикой — чтобы данные в интерфейсе зависели от переменных

Макет:

Открыть макет: https://www.figma.com/design/Ku8wFzBxmPN2D8Ps7KEypt/Test-task-UI-Dev?m=auto&t=jLwZsL9cWnIh0iQq-1
Перейти в режим Inspect для получения детальной информации о элементах на макете

Полезная информация:

Документация для редактора находится в папке QuickEd\Documentation\.
Исполняемый файл редактора для Windows OS: QuickEd\win7\QuickEd.exe
Исполняемый файл редактора для MacOS: QuickEd\mac\QuickEd.app. Если запуск таким образом выдает ошибку, то переместите папку с программой в папку Application и нажимайте Open Anyway на все предупреждения или попробуйте на app правой кнопкой и выбрать Show Package Content, app откроется, как архив, внутри открыть папку MacOS и в ней запустить файл QuickEd.exec и нажимайте Open Anyway на все предупреждения (возможно даже открыть системные настройки и там в разделе Безопасность (Privacy & Security) нажать Open Anyway напротив QuickEd).

Чтобы открыть проект в QuickEd: File → Open Project и выбрать файл ui.quicked в папке TestProject


Пример верстки экрана находится в DataSource/UI/Example. 
Чтобы посмотреть, как экран ведет себя на разных разрешениях нужно изменять параметр Size у корневого контрола.
